FEATURE OVERVIEW
================

Smart Home Platform Support:
- Amazon Alexa (voice commands, ambient sounds)
- Google Home (Cast audio, routine triggers)
- Philips Hue (color temperature, brightness, RGB)
- Generic smart home (extensible architecture)

Story Mood to Lighting Mapping:
- Adventure: Warm golden (HSL 40, 100%, 60%) @ 80% brightness
- Mystery: Deep purple (HSL 270, 80%, 40%) @ 50% brightness
- Happy: Bright yellow (HSL 50, 100%, 70%) @ 90% brightness
- Scary: Blue-green (HSL 180, 70%, 30%) @ 40% brightness
- Calm/Bedtime: Soft blue (HSL 220, 60%, 50%) @ 30% brightness
- Magical: Pink-purple (HSL 300, 80%, 60%) @ 70% brightness
- Sad: Soft blue (HSL 210, 50%, 45%) @ 45% brightness

Ambient Sounds:
- Rain (relaxation)
- Ocean Waves (calming)
- Forest (nature ambience)
- Campfire (cozy)
- Wind (soothing)
- Starry Night (peaceful)
- Birds (morning/awakening)
- Thunderstorm (dramatic)

Bedtime Routine Steps:
- Dim lights (fade to bedtime brightness)
- Play music (lullaby selection)
- Ambient sound (background sounds)
- Read story (start story playback)
- Lights off (complete darkness)
- Voice command (Alexa/Google actions)

DATABASE SCHEMA
===============

smart_home_configs:
- id, userId, platform, deviceName, deviceId

bedtimeRoutines:
- id, userId, childId, name
- scheduledTime, steps (JSON), isActive
- daysOfWeek (JSON), createdAt

API ENDPOINTS (tRPC)
====================

smartHome.getConfig - Get user's connected devices
smartHome.updateConfig - Add/update device config
smartHome.getMoodLighting - Get lighting for story mood
smartHome.triggerLighting - Activate lights for scene
smartHome.triggerBedtimeRoutine - Start bedtime sequence
smartHome.getBedtimeRoutines - List user's routines
smartHome.createBedtimeRoutine - Create new routine
smartHome.getDeviceStatus - Check device connection status
smartHome.disconnectDevice - Disable/remove device
smartHome.enableDevice - Re-enable device
smartHome.getAmbientSound - Get audio URL for mood
smartHome.getAllMoodLightingPresets - Get all mood colors

DESIGN PATTERNS
===============

1. Graceful Degradation
   - API calls fail silently with fallback UX
   - No connected devices shows helpful setup prompts
   - Missing features don't break app experience

2. Modular Architecture
   - Service layer (smartHomeService) handles business logic
   - Router layer (smartHomeRouter) provides API
   - Store layer (smart-home-store) manages client state
   - Component layer provides UI

3. Type Safety
   - Full TypeScript interfaces for all entities
   - Zod validation schemas for API inputs
   - Type exports from database schema

4. Offline Support
   - Zustand store with AsyncStorage persistence
   - Local state cached for offline access
   - Synced when connection restored

5. NativeWind/Tailwind Integration
   - All components use utility classes
   - Consistent spacing and sizing
   - useColors() hook for theme consistency
